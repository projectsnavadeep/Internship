EXECUTIVE SUMMARY
-----------------
InternTrack is a high-performance, enterprise-grade internship application 
tracking platform. It bridges the gap between student workflow management and 
administrative oversight. Built with a "Security-First" and "Aesthetics-Driven" 
philosophy, it utilizes a modern serverless architecture to deliver a seamless, 
Apple-inspired user experience.

1. THE "A TO Z" TECH STACK
--------------------------
Our stack was selected for maximum developer velocity, type safety, and 
UI fluidity.

[FRONTEND]
- CORE: React 19 (Latest stable, utilizing concurrent rendering features).
- BUILD TOOL: Vite 7 (Ultra-fast HMR and optimized production bundling).
- LANGUAGE: TypeScript 5.9 (Strict typing for zero runtime interface errors).
- STYLING: Tailwind CSS 3.4 (Utility-first system for responsive design).
- UI ENGINE: Shadcn/UI + Radix UI (Accessible, unstyled primitives).
- ANIMATIONS: Framer Motion 12 (Used for glassmorphism transitions and reveals).
- CHARTS: Recharts (Data visualization for application funnels and stats).
- ICONS: Lucide React (Clean, consistent vector iconography).

[BACKEND & INFRASTRUCTURE]
- BACKEND-AS-A-SERVICE: Supabase.
- DATABASE: PostgreSQL (Relational integrity for complex application linking).
- AUTHENTICATION: Supabase Auth (JWT-based session management).
- STORAGE: Supabase Storage (S3-compatible buckets for resumes/portfolios).
- EMAIL RELAY: EmailJS (Decoupled email delivery for onboarding/reminders).
- HOSTING: Render (Static site hosting with custom rewrite rules for SPA).

2. CORE SYSTEM LOGIC & ARCHITECTURE
-----------------------------------
This project moves beyond a simple "CRUD" app by implementing professional 
enterprise logic:

- ROLE-BASED ACCESS CONTROL (RBAC): 
  The system identifies users as 'Student' or 'Admin'. Admins gain access to the 
  "Command Center" (Global Analytics, User Registry, and Security Console).
  
- ROW-LEVEL SECURITY (RLS): 
  Security is enforced at the DATABASE level. A user cannot physically query 
  another user's applications even if they have their ID, thanks to Postgres 
  policies checking the `auth.uid()`.

- AUTOMATED ONBOARDING FLOW:
  Logic: [Sign Up] -> [Postgres Trigger] -> [Create Profile] -> [EmailJS Relay] 
  -> [Welcome Email Sent]. This ensures zero "orphan" accounts.

- SINGLETON CLIENT PATTERN: 
  The Supabase client is initialized as a global singleton to prevent 
  authentication state drops during hot-reloads or complex navigation.

- LEVEL-BASED PURGE: 
  A unique HR feature. Level 1 Deletion removes user access while keeping 
  records for audit. Level 2 Deletion performs a full GDPR-compliant data wipe.

3. DATABASE SCHEMA (ENTITIES)
-----------------------------
- PROFILES: Identity, major, university, merit, login stats.
- APPLICATIONS: Company, title, status (Applied, Interviewing, Offer, etc.).
- INTERVIEW_NOTES: Multi-round tracking (Round 1, 2, Final).
- REMINDERS: Deadlines and calendar events.
- DOCUMENTS: Encrypted links to resumes and transcripts.
